# Golang + SQLite 博客

这是一个本地可运行的轻量博客，包含前台展示与后台管理，数据存储在 SQLite 中，支持多用户创建。

## 功能概览
- 前台：首页、归档、文章详情、页面（关于/说说/留言板）、搜索
- 前台认证：用户注册、登录、退出（与后台共用账户体系）
- 后台：登录、文章管理、页面管理、用户管理、留言管理、外观设置
- 访问者后台：独立的 `/member` 后台，可管理“自己的文章/页面/留言/外观/个人设置”
- 内容：Markdown 格式
- 数据库：SQLite（默认 `data/blog.db`）
- 留言板：支持匿名/署名、点赞/点踩，后台可查看 IP、隐藏或删除留言
- 背景：后台“外观设置”提供按钮预设（纯色/渐变/图片），支持本地上传图片并立即生效
- 权限：支持 `站长 / 维护 / 访问者` 三种角色
- 用户管理：站长可修改其他用户的用户名、显示名和角色，可重置其他用户密码；页面可查看最后登录 IP，并记录审计日志
- 头像模块：全站支持用户头像展示；访问者可在个人设置上传头像并提交审核，后台可审核通过/驳回（驳回即不可使用）
- 安全增强：登录限流、同源 CSRF 校验、安全响应头（`X-Frame-Options` / `X-Content-Type-Options` / `Referrer-Policy` / `CSP frame-ancestors`）
- 留言举报：前台可对留言举报，后台“举报中心”可通过/驳回并自动通知相关用户
- 通知中心：访问者后台支持查看通知、单条已读、全部已读
- 会话管理：访问者后台可查看登录会话（IP/设备/时间）并手动下线指定会话
- 账号找回：支持“忘记密码 -> 重置密码”流程（通过通知中心投递重置链接）
- 邮箱验证：注册后可发送邮箱验证链接并完成绑定验证
- 性能增强：公共数据短时缓存 + 可配置慢查询日志（`BLOG_SLOW_QUERY_MS`）

## 前台登录说明
- 访问 `http://localhost:8080/login` 进入前台登录
- 访问 `http://localhost:8080/register` 进行新用户注册
- 前台登录后，导航栏会显示当前用户和退出入口
- 新注册用户默认角色为 `访问者`，可登录前台；是否具备后台管理能力由角色决定

## 启动方式
1. 进入项目目录(保存文件的目录)
   ```bash
   cd path\to\blog
    ```
2. 启动服务（首次启动会自动初始化数据库并生成默认账号）(8080端口,风险较大,可以修改,请自行决定)
   ```bash
   go run ./cmd/blog -addr :8080
   ```
3. 打开浏览器访问(本地运行,如果修改了端口请修改,服务器请访问你的IP地址)
   - 前台：`http://localhost:8080/`
   - 后台：`http://localhost:8080/admin/login`

## 初始化管理员账号
管理员账号可通过环境变量指定：

```bash
set BLOG_ADMIN_USER=admin
set BLOG_ADMIN_PASS=your-strong-password
set BLOG_ADMIN_NAME=站点管理员
```

说明：
- 仓库里不保存默认管理员密码
- 如果未设置 `BLOG_ADMIN_PASS`，程序会在首次启动时自动生成随机密码，并打印到控制台
- 公开仓库前不要提交运行后生成的 `data/blog.db*` 和 `web/static/uploads/`

## 后台使用说明
- 文章管理：新增、编辑、删除文章
- 页面管理：编辑关于/说说/留言板等页面
- 用户管理：创建新的后台用户（多用户支持）

## 密码加密说明
- 当前密码存储方式：`Argon2id`（不可逆哈希，不保存明文密码）
- 存储格式示例：
  - `$argon2id$v=19$m=65536,t=3,p=2$<salt>$<hash>`
- 参数说明：
  - `m=65536`：内存成本（KB）
  - `t=3`：迭代次数
  - `p=2`：并行度
- 兼容策略：
  - 历史 `bcrypt` 哈希仍可登录；
  - 用户成功登录后会自动升级为 `Argon2id` 哈希。

## 慢查询日志
- 默认阈值：`200ms`
- 可通过环境变量覆盖：
  ```bash
  set BLOG_SLOW_QUERY_MS=300
  ```
- 当查询耗时超过阈值时，会在服务日志输出 `slow query` 记录（包含操作名与耗时）

## 迁移提示
后续部署到服务器时，只需要：
- 拷贝整个项目目录
- 如果要迁移现有站点，再额外保留 `data/blog.db` 数据库文件
- 通过 `-addr` 指定监听地址
- 配置反向代理（如 Nginx/Caddy）将域名指向该端口
